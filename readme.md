
webhook-repo
This is a Python Flask application designed to receive GitHub webhook events, store them in a MongoDB database, and display them on a simple, responsive frontend.

Features
Receives push, pull_request, and merge (via pull_request event) webhooks.

Parses relevant data: author, action, from_branch, to_branch, timestamp.

Stores event data in MongoDB with a defined schema.

Exposes a /api/events endpoint to serve the latest events.

Provides a clean and minimal index.html frontend that polls /api/events every 15 seconds to display real-time activity.

Includes basic error handling for webhook payloads and MongoDB operations.

MongoDB Schema
The application stores webhook events in MongoDB using the following schema:

{
  "author": "string",
  "action": "push" | "pull_request" | "merge",
  "from_branch": "string (optional for push)",
  "to_branch": "string",
  "timestamp": "ISO date"
}
